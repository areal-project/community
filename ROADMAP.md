# Roadmap

The authoritative AReaL roadmap is maintained in the **main project repository** at
[areal-project/AReaL](https://github.com/areal-project/AReaL). This file is a pointer
so contributors browsing the community repository can find it easily.

## Where to look

- **Current roadmap document:**
  [`AReaL/ROADMAP.md`](https://github.com/areal-project/AReaL/blob/main/ROADMAP.md)
- **Quarterly tracking issues:** see the
  [issues tagged `roadmap`](https://github.com/areal-project/AReaL/issues?q=label%3Aroadmap)
  in the main repository.
- **Releases:**
  [github.com/areal-project/AReaL/releases](https://github.com/areal-project/AReaL/releases).
- **Release cadence:** minor releases roughly every two weeks, major releases roughly
  every quarter. See the main `ROADMAP.md` for the latest cadence.

## How to influence the roadmap

We welcome community input. The recommended flow is:

1. **Search** existing
   [issues](https://github.com/areal-project/AReaL/issues) and
   [discussions](https://github.com/areal-project/AReaL/discussions) for prior work
   on your idea.
1. **Open a discussion** in the
   [Ideas category](https://github.com/areal-project/AReaL/discussions/categories/ideas)
   to gather feedback.
1. **File a feature request** issue once the idea is concrete.
1. **Indicate support** for existing items with 👍 reactions.

For larger contributions, please coordinate with maintainers before submitting a pull
request — see [CONTRIBUTING.md](CONTRIBUTING.md) and the main project's contributing
guide.

## Long-term vision

Our vision for AReaL is to be the go-to framework for training reasoning and agentic
AI systems that is:

1. **Accessible** — easy to get started for both researchers and practitioners.
1. **Scalable** — runs from a single workstation to multi-thousand-GPU clusters.
1. **Flexible** — supports diverse algorithms, models, and agentic workflows.
1. **Performant** — competitive training speed and efficiency.
1. **Open** — fully open-source with transparent development.
